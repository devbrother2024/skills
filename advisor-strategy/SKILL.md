# Advisor Strategy

상위 모델을 Advisor로 세워 **결정점에서만** 컨설트하는 실행 래퍼.
개념·타이밍 규율·브리프 템플릿의 원본은 이 스킬의 **`references/advisor-strategy.md`** (도구 중립) — 반드시 먼저 읽어라.

## Step 0. 모델 확인과 모드 판별

**advisor 모델은 디폴트가 없다. 사용자가 직접 지정해야 한다.**

1. **모델 파싱**: 인자 첫 토큰이 모델명(`fable`|`opus`|`sonnet`)이면 그것이 advisor 모델, 나머지가 주제다. 사용자 발화에서 모델을 명시했으면 그것도 유효하다.
2. **미지정 시 중단**: 모델이 지정되지 않았으면 **컨설트를 진행하지 말고** 사용자에게 물어라 — "advisor로 쓸 모델을 지정해주세요 (fable | opus | sonnet)". 임의로 모델을 골라 진행하는 것은 금지다 (비용·권한이 걸린 선택은 사용자 몫).
3. **페어링 가드**: 지정 모델이 메인 세션 모델보다 하위 티어면(예: Fable 세션에서 sonnet 지정) advisor 원칙(advisor ≥ executor) 위반임을 알리고 진행 여부를 확인한다. 동급 지정은 유효하다(독립 세컨드 오피니언). 메인 세션 모델의 티어를 확실히 모르면 사용자에게 확인한다.

모드 판별:

| 상황 | 모드 |
|---|---|
| 주제/질문이 지정됐거나 특정 판단이 걸려 있음 | **즉시 컨설트** — 지금 1회 수행 |
| "이 세션을 advisor 전략으로 진행" 취지 | **세션 규율 장착** — 이후 결정점마다 지정 모델로 자동 컨설트 |

## Step 1. 브리프 작성

`references/advisor-strategy.md`의 브리프 템플릿을 채운다. 핵심:

- 사용자 요청은 **원문 그대로** (요약 금지)
- 지금까지의 결정·가정·시도·오류를 결론 없이 사실대로 — **내 접근에 불리한 증거·미해결 의문도 반드시 포함** (브리프 작성자가 executor 본인이라는 편향 완화)
- advisor가 직접 열어볼 수 있는 증거 경로 (파일, diff, 테스트 명령)
- 구체적 결정 질문 1~3개 ("어떻게 생각해?" 금지)
- 조언 길이 제한 줄 포함 (기본 300단어)

## Step 2. Advisor 서브에이전트 호출

컨설트 유형에 따라 에이전트를 고른다. **모델은 Step 0에서 사용자가 지정한 값을 그대로 사용한다.**

| 컨설트 유형 | subagent_type | 비고 |
|---|---|---|
| 접근 확정 전 / 설계·아키텍처 | `Plan` | read-only 플래너. 트레이드오프 판단에 적합 |
| 완료 선언 전 검증 / 막힘 진단 / reconcile | `general-purpose` | 프롬프트에 "파일 수정 금지, read-only" 계약 명시 |

호출 형태:

```
Agent(
  subagent_type: "Plan" | "general-purpose",
  model: "{사용자 지정 모델}",
  run_in_background: false,
  description: "Advisor 컨설트: {주제}",
  prompt: <Step 1 브리프>
)
```

- 컨설트는 블로킹이다(`run_in_background: false`). 조언을 받기 전에 본격 작업을 진행하지 마라.
- **완료 선언 전 컨설트는 산출물을 durable하게 만든 후**(파일 저장/커밋) 호출하라.
- Agent 도구가 없는 하네스(Codex 등)에서는 `references/advisor-strategy.md`의 "하네스별 실행" 절차(`claude -p` 헤드리스)를 따른다.

## Step 3. 조언 처리

1. 조언 핵심을 사용자에게 요약 보고한다 (advisor 모델명 명시).
2. `references/advisor-strategy.md`의 "조언 취급 규칙"을 따른다 — 진지하게 반영, 실증 충돌 시 적응, 내 증거와 조언이 갈리면 조용히 갈아타지 말고 reconcile 컨설트 1회.
3. 조언이 요구사항·범위 변경을 시사하면 구현 전에 사용자 결정을 받는다.

## 세션 규율 모드 (Step 0에서 장착 선택 시)

이후 세션에서 `references/advisor-strategy.md`의 타이밍 규율을 적용한다:

- 본격 작업(쓰기·수정·답 확정) 전 1회 컨설트 — 오리엔테이션은 먼저 해도 된다
- 완료 선언 전 1회 컨설트 — durable 산출물 먼저
- 막혔을 때(오류 반복, 수렴 실패), 접근 변경 시
- 짧은 반응형 턴에는 과호출 금지

장착 시 사용자에게 "이 세션은 advisor 전략 모드로 진행합니다 (advisor: {모델})"를 알린다.

## 경계 (혼동 주의)

- **`/advisor {모델}`은 Claude Code 내장 커맨드**다(네이티브 advisor 설정). 이 스킬은 그것을 **대체하는 독립 구현**이며, 네이티브 존재 여부와 무관하게 항상 서브에이전트 컨설트를 수행한다 (차이점: `references/advisor-strategy.md`의 "참고" 섹션).
- **코드 리뷰가 목적이면 code-review**, **요구사항 대비 구현 검증이 목적이면 verify**를 써라. 이 스킬은 "결정점에서의 방향 조언"이 목적이다.

## 주의

- **재귀 금지** — advisor 서브에이전트가 다시 이 스킬이나 advisor 컨설트를 호출하지 않도록, advisor 프롬프트에는 조언 반환만 요구한다.
- **advisor는 산출물을 만들지 않는다** — 조언을 코드/문서로 옮기는 것은 메인 세션(executor)의 일이다.
- **비용** — advisor 호출은 상위 모델 요율. 결정점에서만 호출하고, 길이 제한 줄을 빼먹지 마라.
