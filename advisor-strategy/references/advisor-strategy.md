# Advisor Strategy — 상위 모델 컨설트 절차 (도구 중립)

> 어떤 코딩 에이전트(Claude Code, Codex 등)에서든 따라할 수 있는 도구 중립 절차다.
> 실행 래퍼는 같은 디렉토리의 `SKILL.md` 참조.

## 개념

Advisor Strategy는 두 모델을 조합하는 에이전트 설계 패턴이다.

- **Executor(실행자)**: 메인 세션 모델. 도구 호출, 파일 수정, 반복 작업을 담당한다.
- **Advisor(어드바이저)**: 더 상위 등급 모델. **결정점에서만** 호출되어 계획·경로 수정·완료 판정 조언을 반환한다. 도구를 직접 쓰지 않고 최종 산출물도 만들지 않는다.

대부분의 턴은 기계적이지만 **계획의 품질이 결과를 좌우하는 long-horizon 작업**에서, 상위 모델을 상시 돌리는 것보다 훨씬 낮은 비용으로 상위 모델급 품질에 근접한다. Anthropic 벤치마크: Sonnet(executor) + Opus(advisor) 조합이 SWE-bench 기준 성능 +2.7%, 비용 -11.9%.

적합하지 않은 경우: 계획할 게 없는 단발 Q&A, 모든 턴이 상위 모델 역량을 요구하는 작업(그냥 메인 모델을 올려라).

## 모델 페어링 규칙

Advisor는 **executor보다 상위이거나 동급**이어야 한다 (Anthropic 공식 원칙. 동급끼리는 서로 advisor 가능 — 독립 세컨드 오피니언). 모델 버전별 허용 조합은 [공식 문서](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool#model-compatibility)를 참조하라.

**advisor 모델에 디폴트는 없다 — 호출할 때 사용자가 직접 지정하고, 미지정이면 컨설트를 진행하지 않는다.** 상위 모델 요율과 접근 권한(Fable 5는 조직 승인 필요)이 걸린 선택이므로 실행자가 임의로 고르지 않는다. 지정 모델이 executor보다 하위면 위 규칙 위반을 알리고 진행 여부를 확인한다. 동급 지정(예: Fable→Fable)이라도 별도 컨텍스트에서의 독립 검토는 가치가 있다.

## 실행 구조 — 서브에이전트 advisor

**상위 모델 서브에이전트를 advisor로 띄운다.** 이 절차는 Claude Code의 네이티브 advisor 도구를 대체하는 독립 구현이다 (네이티브와의 관계는 문서 하단 "참고" 참조).

- executor의 히스토리가 자동 전달되지 않으므로 아래 **브리프 템플릿**을 반드시 채워 전달한다. 브리프 품질이 곧 조언 품질이다.
- 대신 advisor가 **read-only 도구로 저장소를 직접 검증**할 수 있다 — 파일·diff·테스트 결과를 직접 확인하고 조언한다.
- 특정 하네스에 묶이지 않는다 — 서브에이전트나 CLI를 띄울 수 있는 모든 에이전트에서 동작한다.

## 컨설트 타이밍 규율

Anthropic이 실측 검증한 호출 규율이다.

**호출 시점:**

1. **본격 작업 전** — 파일을 쓰기 전, 해석을 확정하기 전, 가정 위에 쌓기 전. 오리엔테이션(파일 찾기, 소스 확인)이 먼저 필요하면 그것을 마친 뒤 호출한다. 오리엔테이션은 본격 작업이 아니다. 쓰기·수정·답 확정이 본격 작업이다.
2. **완료 선언 전** — 호출 전에 산출물을 durable하게 만들어라(파일 저장, 커밋). 컨설트 도중 세션이 끊겨도 durable한 결과는 남는다.
3. **막혔을 때** — 같은 오류 반복, 접근이 수렴하지 않음, 앞뒤가 안 맞는 결과.
4. **접근을 바꾸려 할 때.**

**호출 빈도:** 몇 단계 이상 걸리는 작업이면 최소 2회 — 접근 확정 전 1회 + 완료 선언 전 1회. 직전 도구 출력이 다음 행동을 결정하는 짧은 반응형 작업에는 반복 호출하지 않는다. 조언의 가치 대부분은 접근이 굳기 전 첫 호출에 있다.

**비용 감각:** advisor 호출은 상위 모델 요율로 과금된다. 결정점에서만 호출하면 상위 모델 상시 사용보다 총비용이 낮다. 조언 길이를 제한하면 더 낮아진다(아래 브리프 템플릿의 길이 제한 줄 참조).

## 조언 취급 규칙

- **진지하게 반영하라.** 조언대로 했는데 실증적으로 실패하거나, 1차 소스 증거(파일 내용, 문서 원문)가 특정 주장과 모순되면 적응하라. 셀프 테스트 통과는 조언이 틀렸다는 증거가 아니다 — 테스트가 조언이 짚은 것을 검사하지 않는다는 증거다.
- **증거 충돌 시 조용히 갈아타지 마라.** 내가 찾은 건 X인데 advisor는 Y를 권하면, 충돌을 명시한 reconcile 컨설트를 한 번 더 하라 — "X를 찾았는데 Y를 권한다. 어떤 제약이 결정적인가?" reconcile 한 번이 잘못된 분기에 커밋하는 것보다 싸다.

```
## 계약
- 조언만 반환한다. 파일을 수정하거나 산출물을 만들지 마라.
- 서브에이전트나 advisor 도구를 호출하지 마라. 당신이 최종 조언자다 (재귀 금지).
- 브리프를 그대로 믿지 마라. 브리프의 핵심 전제 중 **최소 하나는 아래 증거 경로를 read-only로 직접 열어 검증**한 뒤 조언하라. 무엇을 확인했는지 조언에 한 줄로 밝혀라.
- 가장 위험한 가정/리스크부터 짚어라. 동의하면 동의한다고 짧게 말해라.
- 조언은 {300}단어 이내로. 포괄 계획이 아니라 집중된 방향 제시가 필요하다.

## 과제 (사용자 요청 원문)
{원문 그대로 — 요약하지 말 것}

## 지금까지의 결정과 가정
- {결정/가정 1 — 근거 포함}
- {내 접근에 불리한 증거·미해결 의문 — 숨기지 말고 반드시 포함}

## 시도한 것과 결과
- {시도 → 결과 (실패했으면 오류 원문)}

## 증거 (직접 확인 가능한 경로)
- {파일 경로, diff 범위, 테스트 명령, 문서 URL}

## 현재 계획
{이대로 진행하려는 접근}

## 질문
{구체적 결정 질문 1~3개. "어떻게 생각해?" 금지}
```

컨설트 유형별 변형:
- **접근 확정 전**: 질문을 "이 접근의 가장 큰 리스크는? 더 단순한 대안이 있나?"로.
- **완료 선언 전**: 계획 대신 산출물 경로와 acceptance criteria를 주고 "빠진 것, 틀린 것, 요구사항 미충족을 찾아라"로. 적대적 검증 프레임이 효과적이다.
- **막힘 진단**: 오류 전문과 시도 이력을 시간순으로. "내가 보지 못하는 원인 가설은?"

## 하네스별 실행

- **Claude Code**: 상위 모델 서브에이전트를 advisor로 스폰한다. 접근/설계 컨설트는 read-only 플래너형 에이전트, 완료 검증·막힘 진단은 범용 에이전트에 read-only 지시. 모델은 사용자가 지정한 값. 상세 실행은 같은 디렉토리의 `SKILL.md`.
- **Codex 등 타 하네스**: Claude CLI 헤드리스 호출로 동일 절차 수행 — 브리프를 파일로 저장 후 `claude -p --model {사용자 지정 모델} "$(cat 브리프.md)"` (권한 없는 read-only 실행이 기본). CLI가 없으면 상위 모델 채팅 세션에 브리프를 수동 전달해도 절차는 동일하다.

## 참고 — Claude Code 네이티브 advisor 도구와의 관계

Claude Code v2.1.98+에는 같은 개념의 네이티브 advisor 도구가 내장돼 있다 (`/advisor` 커맨드, `advisorModel` 설정, `--advisor` 플래그). **이 스킬은 그것을 대체하는 독립 구현**이며, 네이티브 존재 여부와 무관하게 항상 서브에이전트 컨설트를 수행한다. 차이점:

| | 네이티브 advisor | 이 스킬 (서브에이전트) |
|---|---|---|
| 컨텍스트 | 전체 트랜스크립트 자동 전달 | 브리프로 근사 |
| 도구 | 없음 (조언만) | read-only 도구로 저장소 직접 검증 |
| 호출 시점 | 모델이 자율 판단 (상시 규율) | 사용자 명시 호출 (invoke-only) |
| 가용성 | Claude Code + Anthropic API 전용 | 하네스 무관 |

네이티브 advisor는 스킬이나 커맨드 파일이 아니라 Claude Code 내장 기능이다 — `/advisor` 커맨드가 `advisorModel` 설정을 관리하고, API 요청에 서버 도구(`advisor_20260301`)가 붙어 Anthropic 서버에서 실행된다(executor가 호출하면 서버가 전체 트랜스크립트를 advisor 모델에 전달). 이 문서의 "컨설트 타이밍 규율"은 공식 문서의 suggested system prompt를 번역·요약한 것이다.
